# AutoBackup

我现在需要你帮助我构建一个自动备份系统，主要需求如下

## 软件环境
1. 后端 Python 3.14.0 + FastAPI + PostgreSQL + 腾讯云 SDK
2. 前端 Vite + Vue3 + ElementPlus (node 24.15.0)
3. 所有组件容器化，通过docker compose部署 （本机环境是Arm64，你得交叉编译Linux下的x64版本）

## 需求
需要构建一个自动备份系统，分为前端和后端。需求如下
1. 支持创建定时备份任务（支持固定间隔/固定星期，支持设定时间），对服务器的指定目录进行打包（zip格式，支持设置zip密码）
2. 需要支持在管理后台中配置COS存储桶的连接方式，腾讯云账户的凭据等信息。需要支持检测添加的存储桶是否是同地域可以通过内网访问的（通过nslookup检测是否是内网网段），数据库中要保留相关字段
3. 支持将打包产物上传到腾讯云COS存储桶中（定时备份任务配置中需要可以配置上传到指定的存储桶（可同时上传到多个）），上传完成后请删除本地产物以节约本机存储空间
4. 需要支持对COS存储桶中的备份文件进行管理，支持查看和管理所有存储桶中的备份列表（去重后显示，因为上面需求说了备份支持存储到多个存储桶，删除的时候也要一起删除）
5. 需要支持从COS存储桶中下载备份文件进行还原，支持还原到原有目录。对于备份到多个存储桶的，要优先下载内网存储桶，内网存储桶无法访问则弹窗询问是否通过公网存储桶下载
6. 由于COS的公网传输费用很贵，在进行任何从COS下载数据的操作之前都需要检测是否是内网传输，一旦发现不是内网就需要立刻暂停传输作业并询问用户是否继续。上传任务不受限制
7. COS相关凭据信息需要支持加密存储到PostgreSQL数据库中，开发中需要做好安全措施确保凭据不会被泄露
8. 系统管理后台需要支持用户名密码和TOTP/Passkey双因素登录，默认用户名和密码存储在环境变量中，初次进入系统后需要强制修改密码并配置TOTP/Passkey。修改后的用户名和密码需要在数据库中加密存储。
9. 需要有管理员中心的功能，需要支持修改管理员用户名/密码，添加/删除passkey，TOTP
10. 系统需要具有日志功能，需要支持日志的查询和各项操作的记录
11. 系统设计时需要考虑安全、性能和稳定性，管理后台的前端界面使用elementplus组件库，需要操作直观方便，前端界面响应迅速，安全
